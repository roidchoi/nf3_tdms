## § 4. 테스트 케이스

> **구현 Agent에게**: 아래 테스트 케이스를 먼저 코드로 작성한 뒤,
> 모든 테스트가 통과하도록 구현하세요. 테스트 통과 = Task 완료.

### 4.1 정상 동작 케이스

```python
def test_{대상}_{조건}_{기대결과}():
    """
    [목적] {이 테스트가 검증하는 것}
    [유도] {이 테스트를 통과하려면 무엇을 구현해야 하는가}
    """
    # Arrange
    {입력 데이터 준비}

    # Act
    result = {함수 호출}

    # Assert
    assert {기대 결과}
```

### 4.2 경계값 케이스

```python
def test_{대상}_with_empty_input_{기대결과}():
    """[목적] 빈 입력에 대한 처리"""
    ...

def test_{대상}_with_minimum_valid_input_{기대결과}():
    """[목적] 최소 유효 입력에서의 동작"""
    ...
```

### 4.3 예외/오류 처리 케이스

```python
def test_{대상}_with_none_input_raises_value_error():
    """[목적] None 입력 시 ValueError 발생"""
    import pytest
    with pytest.raises(ValueError, match="{에러 메시지 패턴}"):
        {함수 호출}(None)
```

### 4.4 통합/연계 케이스 (해당 시)

```python
def test_{전체 흐름}_{기대결과}():
    """
    [목적] {상위 기능} 전체 흐름 동작 검증
    [유도] 모듈 간 연계가 올바르게 구현되도록 유도
    """
    ...
```

### 테스트 케이스 요약

| # | 테스트명 | 유형 | 검증 내용 |
|---|---|---|---|
| 1 | `test_{...}` | 정상 | {검증 내용} |
| 2 | `test_{...}` | 경계값 | {검증 내용} |
| 3 | `test_{...}` | 예외 | {검증 내용} |
| N | `test_{...}` | 회귀 | {기존 기능 유지} |

**총 {N}개 테스트 — 전체 통과 시 Task 완료**
