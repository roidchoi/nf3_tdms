구현 환경은 지금 작업중인 개발PC와 실제 수집작업을 진행하는 서버PC(window 기반 미니PC 내 wsl2 환경에 배포 및 도커로 실행) 이다.
기능 추가 등을 위해 서버PC의 db와 개발pc의 db를 필요시 동기화 할 수 있는 기능 구현 필요. (개발PC -> 서버PC, 서버PC -> 개발PC, 양방향)
pc이름 또는 ip등을 통한 스스로 어떤 환경인지 파악하는 기능. .env를 통한 각 pc정보 관리. 이를 통해 적절한 환경 설정(db연결정보, api키등) 자동 적용. 
db관련 기능 계획 및 구현시 context7 mcp를 이용해 정확한 사용방법을 학습하고 진행할 것.
기존 경험으로 백업 데이터를 이용한 db 복원 시 인덱스 생성이나 table 생성등의 순서가 틀리는 등의 이유로 복원이 원활히 이루어 지지 않은 경우가 많았음. table추가 인덱스 추가 스키마 수정 등의 다양한 상황에도 db 복원 이전 동기화 등의 문제가 발생하지 않는 강건한 방법 고안 필요. (복원 후 동기화, 점진적 적용 등 다양한 관점에서 검토 필요)
db 연결 오류 등의 최악의 경우를 고려해 실 저장 파일의 경로 등을 반드시 명시하고 해당 경로의 실물 파일의 존재 여부 확인 방법을 코드 내부 또는 관리 문서에 명확히 기술 필요.
재 배포에 따른 도커 이미지 재생성 또는 도커 데스크탑 재 실행 등으로 인한 과정에서 db 가 정확히 연결되고 기존 data가 정상 로드 되는지 스스로 확인하고 문제 발생시 조치 방법에 대한 안내가 될 수 있도록하는 방법 고지기능 필요.
현재 kdms의 db가 유실된 상태이므로 kdms의 db는 기존 db의 복원이 불가할 경우 현재 개발pc의 db상태을 인계받아 누락 data를 backfill 하는 방법을 고려하고 usdms의 db는 정상 수집중으로 서버 pc의 db를 동기화하는 방법으로 진행할 것.
