이 폴더에 tdms 프로젝트의 전체 PRD 문서를 tdms_PRD.md 로 작성할 것.

p1_kdms 는 ref_kdms_wiki 의 backend 기능을 구현하는 것이 목표임 

p2_usdms 는 ref_usdms_wiki 의 backend 기능을 구현하는 것이 목표임 

p3_manager 는 ref_kdms_wiki 의 frontend 기능을 usdms까지 통합 관리하는 기능으로 구현하는 것이 목표임.

현재 각 kdms와 usdms는 수집 작업을 지속 진행 중이므로 기존 수집된 db를 그대로 인계받아 수집작업을 이어서 진행하는 것이 목표임.

p1, p2 는 각각 독립적 구조로 각 기능을 수행할 수 있게 구현하되 kis_wrapper 와 같이 api를 연동하는 부분에 있어서는 공통 모듈을 사용하도록 구현해 접근 토큰 캐시등을 효율적으로 활용할 수 있게 계획할 것.

도커 이미지 빌드 업데이트 시 기존 db data의 유실 사례가 있음. 이에 대한 강건한 예방 구조를 강구할 것. 프로젝트 업데이트 이전 이전 백업 데이터로 복구 등의 전체 과정이 강건히 수행되고 쉽게 관리될 수 있는 구조와 방법으로 구현해야함.

기존에 잘 구현된 기능은 최대한 유지보수 하되 가독성, 유지보수성, 확장성을 최우선으로 고려하여 필요 시 리팩토링, 코드 정제 작업을 적극적으로 수행해야함. 

kdms 와 usdms 간 유사 기능은 서로를 참조해 보다 효율적이고 강건한 구조를 참고 반영해 구현 방향을 가급적 통일해 유지보수의 용이성을 확보할 것

tdms_PRD.md 문서는 각 서브프로젝트와의 관계와 각 서브프로젝트의 관계 등에 중점을 두고 이후 각 서브프로젝트 별 PRD 문서 작성에 참고가 될 정보를 수록하는 것을 목표로 하며 프로젝트별 디테일한 구현 사항은 각 서브프로젝트 별 PRD 문서에 상세히 작성할 것.
